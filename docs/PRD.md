# PRD · KB-Agent（内部 + Partner 共享 Agent 知识平台）

> **文档状态**：本 PRD 覆盖 MVP（M1–M3 + 前端 MF）与已交付的增强批次（F1–F14）。
> 权限模型已从「全局角色 + 空间成员」升级为「全局 admin 绕过一切 + 用户组 RBAC + 空间按组授权」。

## 1. 目标用户与核心场景
| 用户角色 | 使用场景 | 期望结果 |
|---------|---------|---------|
| 管理员（admin） | 管理用户/组/权限、建空间、授权组、维护分类与系统设置、上传管理文档 | 绕过 RBAC 拥有全部权限；集中管控谁能访问什么 |
| 内部员工 | 在被授权空间对话查询、按 RBAC 使用被授权模块 | 拿到「答案 + 原文链接」；仅见被授权的菜单与空间 |
| 外部 Partner | 经用户组被授权访问部分空间 | 只能访问授权空间内容，看不到其他数据 |
| （未来）任意用户 | 通过 skill（如 SCM）借助知识生成 PANW 配置 | 生成配置方案，人工确认后下发 |

**权限模型（现状）**：
- **全局角色** `admin / user`；**admin 绕过一切 RBAC**（migration 012 已合并 internal/partner → user）。
- **用户组**：多维规则（邮箱域名/邮箱/角色 × equals/endswith/contains）自动入组（注册时 + 手动全量重算）。
- **RBAC**：权限绑在用户组上，`模块 × {none/read/write}`；模块 = 对话查询 / 文档管理 / 空间管理 / 用户管理 / 系统设置 / 数据统计 / 新动态（共 7 个）；用户取所属组权限并集最高。
- **空间访问**：个人成员 ∪ 所属组被授权（并存）。

- 前端高级特性（SSR 数据预取优化、URL 路由级 i18n、主题定制）

## 5. 非功能需求
- 性能：面向千级文档；对话走「全空间索引喂 Claude」（上限 `MAX_INDEX_DOCS=200`，超限按最近截断并记日志）。
- 安全：JWT 锁 HS256；workspace 强制隔离（个人∪组）；密钥走 `.env`/云 secret，禁止硬编码；下载 attachment+nosniff；LLM 产物前端净化渲染（防 XSS）。
- 可维护性：LLM 唯一出口 `engine/`、文件唯一出口 `storage/`；api 层无业务逻辑；每端点配 pytest（happy + error）；测试用独立库 `kbagent_test`，不污染 dev 数据。
- 可扩展性：引擎多后端可切换（Claude 可用，Codex/OpenClaw 预留并前端灰显）；skill 框架抽象预留；分类/目录层级可扩展。
- ⚠️ 安全取舍：Claude CLI 放开全部工具（含 Bash），容器以非 root 运行；提示词注入由外部 Guardrails 负责（见 SECURITY.md #1/#2）。
